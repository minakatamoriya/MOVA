# 守护核心模式终局结构与精英掉落设计

本文档用于细化守护核心模式的终局结构，以及精英掉落在新模式中的职责与第一版掉落池方向。

本文档建立在以下前提之上：

- 玩家成长采用主动升级。
- 金币不直接恢复核心生命值。
- 核心生命恢复主要通过部件、模块或条件触发实现。
- 精英掉落不应直接替代核心模块购买决策。

---

## 1. 终局目标

当前推荐的终局不是：

- 开局就公开固定倒计时
- 时间到直接判定胜负
- 用秒杀式机制强行收局

当前推荐的终局是：

- 表面体验是软无尽
- 中后期进入不可逆的终局攻城
- 玩家最终通常会在高压下被逐步压垮

这种结构的目标是让玩家感受到：

- 前期是在经营和构筑
- 中期是在拉扯和补线
- 后期是在和不可逆浪潮对抗

---

## 2. 为什么采用“软无尽 + 必然崩盘”

这种结构比固定 20 分钟更适合当前模式，原因包括：

- 更符合守基地而不是跑倒计时的体验
- 高潮来自局势崩坏，而不是时间结束
- 玩家更容易接受失败，因为失败来自撑到极限，而不是被系统处决
- 可以保留导演系统的节奏控制，又不必让玩家始终盯着计时器

---

## 3. 终局分阶段结构

### 3.1 第一阶段：建立期

玩家体验：

- 规则逐渐清晰
- 金币开始稳定流入
- build 开始形成基础轮廓

导演目标：

- 让玩家理解前线打钱和守核心的关系
- 让玩家体验第一次主动升级与第一次核心布置

### 3.2 第二阶段：拉扯期

玩家体验：

- 前线收益开始变得诱人
- 核心压力开始要求回看局势
- 玩家必须开始做取舍，不能什么都兼顾

导演目标：

- 引入更稳定的漏怪压力
- 让精英开始承担打破稳态的职责
- 让玩家形成前压、回看、再前压的循环

### 3.3 第三阶段：崩盘前夜

玩家体验：

- build 已经接近成型
- 但敌方增长速度开始追上甚至反超玩家效率
- 玩家能明显感觉到“局势开始不再可完全控制”

导演目标：

- 提高精英出现频率
- 提高特殊敌人叠加出现概率
- 让核心区域压力从偶发转为常态化

### 3.4 第四阶段：终局攻城

玩家体验：

- 目标从稳态经营转为极限多撑一会
- 前线无法再被完全守住
- 核心区域成为长期主战场

导演目标：

- 持续提高破线压力
- 让精英不再只是插曲，而成为常驻压力源
- 迫使玩家用已成型 build 做最后抵抗

---

## 4. 玩家看到的前台表现

当前推荐：

- 不把固定倒计时作为前台主信息
- 可以把导演进程包装成“威胁等级”或“攻城等级”
- 同时把时间显示改成“已坚守时间”或“本局存活时间”，而不是“剩余通关时间”

例如可采用：

1. 威胁等级 1 到 3：建立期
2. 威胁等级 4 到 6：拉扯期
3. 威胁等级 7 到 8：崩盘前夜
4. 威胁等级 9 到 10：终局攻城

这样玩家看到的是局势升级，而不是简单计时器。

进一步的正式口径建议为：

- 前台主信息显示已坚守时间。
- 前台同时显示威胁等级、攻城阶段或局势警报。
- 局后奖励按坚持分钟数与局后评价共同结算。
- “20 分钟”不再作为玩家必须看到的固定导演目标，只保留为后台参考窗口。

---

## 4.1 动态导演而不是固定 20 分钟脚本

当前更推荐把导演理解为：

- 难度会持续递进
- 但不是严格按 20 分钟脚本线性播放
- 而是根据战局表现做动态推演

也就是说，时间仍然可以存在，但只作为弱参考，不应继续做成唯一主轴。

当前更推荐的动态导演输入包括：

1. 已坚守时间
2. 当前威胁等级
3. 核心近区承压频率
4. 漏怪频率与前线失守频率
5. 精英存活时间与处理效率
6. 玩家清怪效率与金币成长速度

推荐的控制口径是：

- 时间负责给导演一个最低递进基线。
- 战局表现负责决定当前应该平稳推进、提前加压，还是短暂给喘息窗口。

进一步的正式口径建议为：

- 底层持续刷怪不停止。
- 上层通过峰值波制造一轮轮主要攻势。
- 峰值波完成后，允许进入短缓冲窗口。
- 缓冲窗口不停刷，只回落到低压档。
- 很多峰值波的结束条件可以依赖该轮峰值精英全部死亡。
- 玩家可以在任何时间点开升级，但缓冲窗口最适合回核心做升级和重整。
- 点开升级后，第一版采用全局暂停，确保完整决策空间。

这样做的好处是：

- 玩得强的局不会因为固定时间轴而显得呆板。
- 玩得弱的局不会因为时间一到突然被脚本硬压死。
- 每一局都更像“撑到极限”，而不是“背一条固定流程”。

同时，这套结构还能自然形成：

- 打完一轮主要攻势
- 回核心处理成长与模块选择
- 再回前线准备下一轮高峰

的完整循环。

## 5. 终局必须避免的风险

### 5.1 不能突然处决玩家

如果终局没有预警，而是突然把怪量或伤害暴力拉满，玩家会觉得不公平。

因此终局前必须有清晰的预警阶段。

### 5.2 不能只靠刷更多怪

如果终局只是怪越来越多，而怪物结构没有变化，体验会很快变得单调。

因此终局必须同时提高：

- 敌人数量
- 敌人种类叠加
- 精英存在率
- 漏怪与破线风险

同时还应补一条：

- 怪物血量主曲线应由导演阶段推进，但不能靠把所有怪都变成高血木桩来制造压力。

正式建议为：

- 基础潮怪尽量保持数量感和割草感。
- 厚皮怪、精英和峰值目标承担更多血量增长职责。
- 后期的紧张感主要来自数量、组合、破线风险和持续承压，而不只是单体血量膨胀。

### 5.3 不能让后期只剩数值硬顶

终局要保留“还能再争取一点”的感觉。

因此后期仍需要：

- 可做的升级决策
- 可影响局势的核心模块
- 明显可见的最后挣扎空间

---

## 6. 精英掉落的职责

在当前新模式里，精英掉落不只是奖励，更承担以下职责：

- 强化前线收益的价值
- 让玩家对精英产生明确优先级
- 在中后期持续改变局势处理方式
- 为核心模块购买提供阶段性助力，但不直接替代购买决策

因此精英掉落不应只是：

- 多给一些金币
- 给一件泛用旧装备

而应尽量掉出能改变后续决策权重和资源节奏的奖励。

---

## 7. 第一版精英掉落主池

当前建议第一版优先做以下四类掉落：

1. 前线赏金模块
   - 作用：提高前线击杀收益，强化前压打法

2. 精英猎手战利品
   - 作用：提高处理精英的效率，帮助玩家在后期稳定清除关键威胁

3. 核心采购券
   - 作用：降低下一次核心模块购买价格，但不直接送核心等级

4. 战备拨款
   - 作用：提供一笔立即可用的共享金币，保证击败精英后能立刻做一次主动投入

这四类更适合作为第一版精英奖励主池，因为它们不会直接绕过金币升级和购买决策。

进一步建议把第一版精英收益拆成三层，而不是只做单层随机表：

1. 直接金币层
   - 用于保证击败精英后立刻有一次可感知消费。
   - 这层主要承担现金流修复，不承担流派定义。

2. 决策增强层
   - 掉下后立刻影响接下来的投入方式。
   - 例如核心采购券、专项金币补贴、前线收益提高。
   - 这层负责“马上改变接下来 1 到 3 分钟的决策权重”。

3. 解锁资格层
   - 不直接生效，但解锁新的购买权限、槽位或模块分支。
   - 这层更适合中后期，避免所有成长都挤在一次掉落里瞬时完成。

这三层组合的目的，是避免以下两个问题：

- 只有金币，导致精英优先级不足
- 只有直接功能件，导致核心太早成型、购买决策被运气替代

### 7.1 第一版推荐掉落结构

当前更推荐的落地方式是：

1. 每只精英固定掉一笔较高价值金币
2. 同时再掉一个结构奖励
3. 少量精英改为掉“购买许可”或“槽位许可”

也就是说，第一版不要把精英收益做成“金币和结构件二选一”。

更稳的方案是：

- 金币负责让玩家马上能消费
- 决策增强负责改变 build 走向
- 许可负责延迟成长上限

### 7.2 掉落来源建议

建议不要让所有精英共用完全同一掉落表，而是至少按职责分三类：

1. 前线追击型精英
   - 更偏掉前线赏金、精英猎手、距离收益类奖励

2. 近核施压型精英
   - 更偏掉核心采购券、战备拨款和回防专项奖励

3. 峰值波核心精英
   - 更偏掉高价值金币包、槽位许可、稀有结构件

这样做的好处是：

- 玩家会根据当前缺口判断该优先处理哪种精英
- 掉落结果会反过来强化“怪物职责辨识度”
- 精英不会退化成只是数值更高的同一种怪

### 7.3 阶段化掉落建议

第一版建议按导演阶段解锁掉落层级：

1. 建立期
   - 主要掉基础金币和低阶结构件
   - 目标是尽早形成第一次明确 build 变化

2. 拉扯期
   - 开始稳定掉前线收益类和核心防线类奖励
   - 目标是让玩家开始真正做前压与回防取舍

3. 崩盘前夜
   - 引入槽位许可、模块强化件、精英专杀类奖励
   - 目标是让中后期精英真正成为改变局势的节点

4. 终局攻城
   - 低阶奖励占比下降，金币包和高阶许可占比提高
   - 目标是让后期更像“最后压榨 build 极限”，而不是无限白送组件

### 7.4 市面上可借鉴的方向

可以参考以下几类成熟做法，但不建议原样照搬：

1. Vampire Survivors 的宝箱节点
   - 优点是击败关键目标后会立刻迎来强反馈和 build 跃迁。
   - 借鉴点是：精英奖励应该像节奏节点，而不是普通掉落放大版。

2. Brotato 的精英压力
   - 优点是精英存在的价值首先来自战场压力，而不是只来自奖品。
   - 借鉴点是：先让精英成为必须处理的问题，再让奖励强化这一优先级。

3. Hades / Death Must Die 一类的构筑奖励
   - 优点是奖励标签非常明确，玩家能快速理解这次 build 正在往哪里偏。
   - 借鉴点是：每个精英掉落最好都有清晰流派标签，不要做成大量模糊小词条。

守护核心模式更适合吸收它们各自的一部分：

- 用 Vampire Survivors 的“关键节点强反馈”
- 用 Brotato 的“压力优先于奖励”
- 用 Hades / Death Must Die 的“奖励标签清晰、容易形成流派”

而不适合直接照搬的部分包括：

- 直接送核心模块等级
- 复杂背包管理
- 奖励稀有度和词条层数过多

### 7.5 第一版推荐的具体掉落框架

如果要进入可开发状态，当前建议先固定为下面这套：

1. 每个压阵精英
   - 固定掉高价值金币
   - 再从精英奖励池中掉 1 项

2. 每个阶段首个精英
   - 有更高概率掉许可型奖励

3. 前线击杀精英
   - 额外提高金币包价值
   - 更高概率掉前线收益类奖励

4. 近核击杀精英
   - 更高概率掉核心采购券或战备拨款

5. 终局攻城阶段精英
   - 提高金币包和稀有许可权重
   - 降低基础经济奖励的重复率

这套结构的好处是：

- 规则简单，前台容易说明
- 精英收益不再和普通怪完全同质
- 可以在不引入复杂背包的前提下做出 build 分化

---

## 8. 第一版不建议接入的掉落

当前第一版不建议优先接入：

- 一次性直接恢复核心生命的大药包
- 大量围绕玩家本体生存的旧式装备
- 复杂背包管理
- 大量玩家战利品和核心模块同时铺开

这样做是为了避免系统数量过快膨胀。

---

## 9. 与局后评价的关系

精英掉落和终局结构应共同影响玩家的局后评价。

推荐重点关注：

- 玩家是否主动处理精英
- 玩家是否利用精英掉落强化守家结构
- 玩家是否在终局攻城阶段把 build 发挥到了极限

这会让精英不再只是掉宝怪，而是真正成为整局节奏的关键节点。
