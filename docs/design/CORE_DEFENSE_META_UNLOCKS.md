# 守护核心模式局外解锁与非数值成长

本文档用于定义守护核心模式的局外成长边界、解锁货币、结算逻辑与长期目标设计。

本文档默认前提：

- 局内成长是玩法主轴。
- 局外成长不应破坏局内公平与 build 决策。
- 局外内容当前收敛为纯展示型解锁，而不是永久数值压制。

---

## 1. 是否需要局外成长

当前结论：

- 需要局外成长。
- 但当前不做局外数值成长。

原因如下：

- 完全每局从零开始虽然纯粹，但长期留存偏弱。
- 纯数值局外成长虽然容易拉长养成，但会明显侵蚀局内 build 决策价值。
- 守护核心模式当前最值得保留的，是局内资源分配与临场判断，而不是“号养大了自然赢”。

因此当前更推荐的方向是：

- 局外做纯展示型成长
- 局内做重决策

---

## 2. 局外成长的边界

### 2.1 优先做什么

当前最适合放进局外系统的内容包括：

- 玩家皮肤
- 核心皮肤
- 特殊演出或装饰性奖励
- UI 主题、边框、徽记等展示物
- 图鉴与展示类内容

这些内容的共同特点是：

- 能提供长期目标
- 不直接破坏局内数值公平
- 允许玩家积累进度感

当前先不把地图、关卡、难度作为局外成长主轴，避免再次把局外解锁引向玩法数值层。

### 2.2 暂不优先做什么

当前不建议优先做的局外内容包括：

- 永久伤害加成
- 永久金币倍率
- 永久核心生命大幅提升
- 永久核心恢复效率大幅提升
- 大量起始战力装备直接带入局内

这些内容会让局内玩法越来越依赖号强度，而不是单局操作与资源决策。

---

## 3. 局外货币设计

### 3.1 不建议直接用剩余金币结算局外成长

当前不推荐让局外解锁直接依赖“本局剩余金币”。

原因如下：

- 会让玩家不敢花金币
- 会削弱升级与部件部署的正当性
- 会把局内主资源和局外主资源绑死
- 会诱发“省钱比打好更重要”的反常玩法

因此建议明确分离：

- 局内金币只属于本局
- 局外解锁使用独立结算货币

### 3.2 推荐独立局外货币

当前建议引入独立的局外结算资源，例如：

- 守护点数
- 守护徽记
- 战后评定点数

名称后续可再定，但逻辑应保持：

- 不是金币
- 不参与局内消费
- 只用于局外解锁

---

## 4. 局后结算逻辑

### 4.1 推荐结构

当前最推荐的结算结构为：

局外点数 = 基础存活奖励 + 表现奖励 + 里程碑奖励

这套结构的好处是：

- 每局都有保底积累
- 打得更好会更快解锁
- 不会过度鼓励省钱或苟活

### 4.2 基础存活奖励

基础奖励建议按以下要素提供：

- 存活分钟数
- 达成的局势阶段
- 是否进入终局波段

基础奖励的意义是保证：

- 就算失败也有稳定进度
- 新手不会因为连续失败完全零回报

### 4.3 表现奖励

表现奖励建议优先绑定守护核心模式的核心行为，而不是泛泛地绑定输出总量。

适合的指标包括：

- 击杀精英数量
- 核心剩余耐久比例
- 漏怪控制情况
- 前线击杀占比
- 核心承压控制情况

这里的作用是：

- 鼓励玩家认真守家
- 鼓励玩家前压而不是长期缩线
- 避免系统只奖励苟活

### 4.4 里程碑奖励

里程碑奖励用于提供中长期目标。

可参考方向：

- 首次撑到某个时间节点
- 首次击杀某类精英
- 首次将任一技能升到五档
- 首次在高耐久状态下完成某阶段
- 首次安装多个核心模块

这类奖励适合直接对应：

- 皮肤解锁
- 核心皮肤解锁
- 特殊演出解锁
- 展示徽记解锁

---

## 5. 评级系统是否需要

当前可以有评级系统，但推荐把评级放在展示层，而不是把评级作为唯一的资源计算基础。

也就是说：

- 底层仍然按存活、表现、里程碑结算点数
- UI 上可以再给出 S / A / B / C 等评级

评级更适合承担：

- 局后成就展示
- 展示类奖励门槛
- 稀有外观门槛

而不是完全替代实际结算逻辑。

---

## 6. 当前公平性结论

由于局外只做展示型解锁，当前默认所有局内数值规则保持一致。

也就是说：

- 不依赖局外数值成长提供优势
- 不需要用额外挑战模式来兜底数值公平
- 单局公平性天然由系统边界保证

---

## 7. 推荐的第一版局外系统

如果只做一版最小局外系统，当前建议优先实现：

1. 独立局外货币
2. 按存活时间结算基础点数
3. 按精英击杀和核心状态补表现点数
4. 少量里程碑解锁玩家皮肤、核心皮肤与展示物
5. 暂不接入永久数值成长

这个版本已经足够支持：

- 长期目标
- 不同玩家的收集欲
- 新内容逐步开放

同时不会破坏局内成长系统的核心地位。
