# 天赋系统复盘

本文档不是技能表，而是对当前版本天赋系统的结构问题做一次面向实现的复盘。结论基于当前代码，不基于历史设想。

## 结论摘要

当前版本的主问题不在“有没有内容”，而在“内容如何被组织和呈现”。

- 主职业身份已经建立起来了，但职业间完成度不均。
- 副职业主题已经比旧版清晰，但不同分支的即时反馈强弱差异很大。
- 深度专精已经接回主流程，但解锁时点偏晚、说明不够透明。
- 天赋文案来源分散，导致代码、树描述、升级卡和文档极易漂移。
- 很多节点虽然名义上压缩成了 1 级或 2 级，但底层仍沿用旧的 3 档内部数值，维护成本偏高。

---

## 一、职业设计问题

### 1.1 职业完成度不均衡

现在六个主职业都能成立，但成立方式不一致。

- 法师、猎人：核心循环最清晰。玩家能快速感知“叠层爆裂”或“箭列成长”。
- 德鲁伊、术士：主题成立，但信息密度高，很多强度来自多段联动，玩家不容易一眼读懂 build 关键点。
- 战士：基础身份明确，但主树节点过于朴素，和深度专精之间有明显割裂。
- 圣骑士：体系丰富，但主职业、守护副职业、深度专精之间的边界感偏模糊，很多表现都落在“反击/脉冲/审判”这一团主题里。

问题不是题材不够，而是“玩家在前 3 到 5 次升级里，能否迅速形成这个职业的战斗语言”。目前只有部分职业做到了这一点。

### 1.2 主职业与副职业的即时反馈不均衡

副职业入口虽然统一成了事件选择，但选中当下的战场收益并不对齐。

- summon：直接拿到 1 卫士 + 1 法师，立刻有很强的战场存在感。
- guardian：直接拿到格挡和圣印，也能立刻感知防御层变化。
- nature：直接拿熊灵，也有明显体感。
- arcane、ranger、unyielding：虽然会开启对应系统，但当下的反馈没有 summon / guardian / nature 那么强。

这会让副职业事件看起来像“并不是六选一，而是几条分支天然更赚”。如果不做统一，玩家对副职业平衡的第一印象会长期失真。

### 1.3 生存节点的题材区分度不足

很多职业都采用“生命低于 30% 时自动触发”的生存节点：

- archer_nimble_evade
- warrior_blood_conversion
- mage_frost_nova
- paladin_divine_shelter
- druid_nourish
- warlock_infernal

这套模板很好用，但用得太多之后，职业之间的保命方式就会越来越像，只剩数值差异，没有策略差异。

更好的方向不是全部删掉，而是至少让其中一部分职业把保命从“低血自动触发”改成更贴合职业身份的触发逻辑。

## 二、天赋结构问题

### 2.1 主树结构已统一，但分段职责还可以更鲜明

当前六个主职业主树已经统一成 0 / 3 / 6 三段式，这一步是必要的，因为它至少让所有职业都在使用同一种树语言。

但统一门槛不等于统一体验，后续仍然有两个问题：

- 有些职业在 3 点段拿到的是明确功能节点，有些职业拿到的仍然更像数值补丁。
- 深度专精虽然都在 6 点开启，但并不是每个职业都能形成同样鲜明的“后期高光时刻”。

所以现在的问题已经从“结构不统一”转成了“统一结构下，各段职责是否清晰”。

### 2.2 一部分节点仍然是“纯数值卡”

当前压缩过后已经比以前好，但仍然有不少节点主要只是在加数字：

- mage_frostbite
- mage_cold_focus
- mage_ice_veins
- guardian_armor
- summon_necrotic_vitality
- druid_pet_bear / hawk / treant 的首级强化描述也偏数值入口

纯数值卡不是不能存在，但如果它们出现在前中期高频候选位，就会稀释 build 选择感。玩家会感觉是在补 KPI，而不是在做路线决策。

### 2.3 压缩后的运行时模型仍带着旧三档影子

很多节点在文档上是 1 级或 2 级，但 runtime 里仍然会直接把内部等级写成 2 或 3。比如：

- 单级节点常直接把内部 level 设为 3
- 二级节点常直接从 0 跳到 2，再跳到 3

这能兼容旧数值表，但代价是：

- 文案难维护
- 调数值时容易错把内部 level 当玩家等级
- 树描述、卡面描述、实际表现很容易再次分叉

这已经不只是文档问题，而是数据模型问题。

### 2.4 前置关系设计不统一

有些分支的依赖链很清楚，比如 arcane、guardian、nature、ranger；有些主职业则几乎完全平铺，比如 mage、warrior。

结果是：

- 部分职业的 build 形状很明确，玩家知道要先拿什么再拿什么。
- 另一部分职业更像“数值自助餐”，顺序影响较弱。

如果这是有意为之，需要明确职业差异；如果不是，就说明主树的结构语言还没统一。

## 三、升级节奏问题

### 3.1 副职业事件触发点固定，但主树建立度不一定足够

当前副职业选择在普通升级次数到 3 时触发。这个阈值的优点是稳定，但它默认假设：

- 所有主职业都能在前 3 次升级里完成“核心玩法确认”

实际并非完全如此。

- 猎人、法师通常没问题。
- 圣骑、术士、德鲁伊有时还没来得及让主树建立清晰手感，就已经进入副职业选择。

这会让玩家在主职业还没站稳时，就被迫做第二路线决策。

### 3.2 深度专精开启偏晚，而且条件不透明

深度专精同时受三层约束：

- 普通升级阶段必须进入 all
- 主树至少 6 点
- 副树至少 2 点

这个设计本身合理，但对玩家完全不透明。结果是：

- 玩家不知道为什么深度节点还没出现
- 玩家无法主动规划“离深度还差什么”
- 深度专精作为高光内容，登场时机反而容易显得突兀或滞后

这类问题最好通过 UI 提示或树内门槛显示解决，而不是只藏在发牌逻辑里。

### 3.3 发牌权重更偏稳定推进，而不是高光节点展示

现在发牌权重的方向是合理的，但结果是中期较保守：

- 主职业在前期权重很高
- 副职业有追赶权重
- 深度专精基础权重只有 0.42，而且未满足条件前还会被大幅压低

这让系统更稳定，但也会带来一个副作用：

- 中盘的“哇，这把 build 开始起飞了”时刻来得偏慢

如果游戏更偏爽感，这种权重策略可能过稳。

## 四、文档与数据源问题

### 4.1 同一节点有三套描述源

目前至少存在三类文本来源：

- talentTrees：树上概述文案
- upgradePools：池子简述
- upgradeOfferPresentation：逐级卡面文案

再加上技能总表，很容易形成四份口径。

这次重写文档能缓解，但不能根治。根治方式应该是把节点元数据统一成单一来源，然后由 UI 按不同场景裁剪展示。

### 4.2 代码里还保留不少历史兼容项

比如旧 id alias、旧等级映射、旧技能名残留。这些兼容项是必要的，但如果没有一个明确的“现行口径文档”，维护时就很容易误用旧名词。

这也是为什么文档会越来越难看：不是没人写，而是底层对象本身有历史包袱。

## 五、按分支看的具体不足

### 5.1 战士

- 主树过于直给，前中期选择张力不足。
- 深度专精比主树更像“真正的职业高潮”，导致前后体验断层。

### 5.2 法师

- 主树结构清楚，但太多单点数值卡集中在一起，导致早期可选项像在补面板。
- mage_shatter 是真正的玩法节点，强度存在感远高于同排其他点。

### 5.3 圣骑士

- 主题丰富但边界感弱，很多内容都围绕“防守反击”展开，树内分工不够清晰。
- paladin_avenger 的定位还不够独立。

### 5.4 术士

- 主题很强，但毒沼、吸血、召唤、地狱火都在一个职业语义里，阅读负担偏高。
- 主树和深度专精都偏滚雪球，前中后期的节奏分层没有猎人、法师那么清楚。

### 5.5 德鲁伊

- 主树、自然伙伴、深度专精都在强调“覆盖 + 伙伴 + 连锁”，后期容易视觉和逻辑同时过载。
- druid 主树点数偏长，但前中期的决定性节点不够突出。

### 5.6 猎人

- 是当前最接近完整范式的一支，但副职业围猎线的强度和乐趣有一部分被 summon、guardian 的入口反馈压住了。

---

## 六、优先级建议

如果只做一轮中等规模调整，我建议优先处理下面 5 件事：

1. 统一节点数据源，把树描述、卡面描述、技能文档改成同一份节点元数据导出。
2. 给深度专精增加可见解锁提示，至少明确显示“主树 6 点 / 副树 2 点 / 后段阶段”。
3. 校准副职业入口即时反馈，让 arcane、ranger、unyielding 的选中瞬间不显著弱于 summon、guardian、nature。
4. 在已经统一的 0 / 3 / 6 结构上，继续明确各职业三段分别承担什么职责，避免有的职业第二段仍像纯数值补丁。
5. 继续减少前中期纯数值卡，把更多节点改成“改变战斗行为”而不是“只提高数值”。

## 七、长期方向建议

如果后面要继续打磨这一套系统，长期上更值得做的是：

- 主职业前 3 次升级必须保证至少出现 1 个真正改变玩法的节点，而不是只给数值补丁。
- 副职业入口统一成“选中当下就能看懂收益”的即时反馈设计。
- 深度专精做成明确的后期高光段，避免只是另一组数值卡。
- 把所有等级和描述都收敛到单一 schema，减少文案漂移和历史映射成本。

当前系统已经能玩，但还没有完全到“每个选择都像在塑造 build”的状态。下一阶段最值钱的工作，不是继续往里塞新天赋，而是把已有结构打磨到更清晰、更可读、更有选择张力。
